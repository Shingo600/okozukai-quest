# レビューチェックリスト

実装後、Claude Code が自己レビューする項目。

## 機能の整合性

- [ ] 既存機能を壊していないか（呼び出し元・関連画面を確認）。
- [ ] 型の整合性が保たれているか（`tsc --noEmit` が通る）。
- [ ] エラー時の挙動が破綻していないか（例外を握りつぶしていないか）。

## UI / 表示

- [ ] UI 変更がある場合、表示崩れ・レイアウト破綻がないか。
- [ ] 既存スタイルとの整合性は取れているか。

## データ互換性

- [ ] 保存データ・既存データとの互換性があるか。
- [ ] スキーマ・キー名を破壊的に変更していないか。

## 自動チェック

- [ ] `npm run harness:check` が通ったか。
- [ ] 必要に応じて `npm run build` が通ったか。

## 報告

- [ ] 変更ファイル・確認結果・残リスクを日本語で要約したか。
