# 実装チェックリスト

実装前後に Claude Code がセルフチェックする項目。

## 着手前

- [ ] 依頼内容の対象範囲（どこまで触るか）を理解したか。
- [ ] 変更前に関連ファイル（型・呼び出し元・近接コンポーネント）を読んだか。
- [ ] 不明点はユーザーに確認したか（推測で進めていないか）。

## 実装中

- [ ] 型定義の更新が必要な場合、先に型を更新したか。
- [ ] 既存の構成・命名・スタイル・CSS 規約を優先したか。
- [ ] 既存ユーティリティ・コンポーネントを再利用したか（重複実装を避けたか）。
- [ ] 不要な依存関係を追加していないか（追加した場合は理由を明記）。
- [ ] 大規模リファクタリングを勝手に行っていないか。

## セキュリティ観点

- [ ] API キー・トークン・パスワードを直書きしていないか。
- [ ] 機密情報を `console.log` していないか。
- [ ] 不要な外部通信を追加していないか。
- [ ] `eval` / `new Function` を使っていないか。

## 仕上げ

- [ ] `npm run harness:check` を実行し、通過したか。
- [ ] 本番影響がある場合、`npm run build` を実行したか。
- [ ] 変更ファイル一覧・残リスクを日本語でまとめたか。
